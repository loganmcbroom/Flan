Detailed Description

AudioBuffer stores audio data and provides basic buffer access, conversion constants, loading, and saving. Access to the raw sample buffer is given, but AudioBuffer::getSample and AudioBuffer::setSample are preferred when speed is not a factor. Data is stored in channel-major order, in other words, the entire first channel is stored in memory before the second channel's storage begins.

A note on terminology, the term "frame" is used to describe points in time at which samples are taken. The term "sample" is used to describe the actual data at that point in time.
