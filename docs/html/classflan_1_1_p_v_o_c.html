Detailed Description

Phase Vocoder (PVOC) Data Algorithms.

This is a stateless wrapper of flan::PVOCBuffer which contains all relevant algorithms and utilities available in flan for manipulating PVOC buffers.

All methods are const and have no side effects outside of PVOC::graph, which saves a bmp file. See PVOCBuffer for a description of Phase Vocoder (PVOC) Data.

This is nearly identical to PVOC::modify_cpu. Due to the computations being on the gpu in parallel it is much faster. The cost of this speed is that the output buffer is write only, so any output bin mapped to more than once will take one of those written values only, with no determination as to which. This method will return the input if flan was built without OpenCL.

◆ modify_cpu()

PVOC flan::PVOC::modify_cpu ( Func2x2  mod,
Interpolator  interp = Interpolators::linear,
flan_CANCEL_ARG   
) const

Each input bin is mapped to an arbitrary output point by mod. Every quad in the input is thus mapped to a quad in the output. The output points within that quad are given an average of any data already within the bin and the four input bins that made up the quad being mapped. Quad averages are decided using the interpolator argument and the algorithm described here: https://www.particleincell.com/2012/quad-interpolation/

Parameters
modTakes and returns time/frequency pairs as either flan::vec2 or std::complex<float>
interpInterpolator used in quad mapping

Each bin has its frequency copied to subsequent output bins with decaying magnitude until a bin with magnitude greater than the current decayed magnitude is read from the input. The frequency of the output at that point then changes to that of the louder bin.
